# MicroNT System Service Dispatch

*Implementation note (MicroNT, Apr 2026).*

---

This note covers the path a system service call takes from user or kernel
mode through the dispatcher and into the service routine. NT 3.5 originally
routed every call through `INT 2Eh`; MicroNT replaces that with `SYSENTER` /
`SYSEXIT` for user mode and a direct in-image call for kernel mode.

The two paths share nothing at the trap-frame level any more — the dispatch
body in `KiSystemServiceShared` is reached only from the SYSENTER entrypoint;
kernel-mode `Zw*` calls bypass it entirely.

## 1. Background: the INT 2Eh path (original NT 3.5)

Every system service call — `Nt*` from user mode, `Zw*` from kernel mode —
went through the same gate-trap mechanism:

1.  The user- or kernel-mode stub (generated by `gensrv` from
    `KE/I386/SERVICES.STB`) loaded `eax = ServiceNumber` and
    `edx = &args`, then issued `INT 2Eh`.
2.  The CPU dispatched through IDT vector 0x2E to `_KiSystemService`.
    For user→kernel transitions it switched stacks via `TSS.Esp0` and
    pushed `SS / ESP / EFLAGS / CS / EIP`; for kernel→kernel it pushed
    only `EFLAGS / CS / EIP` (no stack switch).
3.  `ENTER_SYSCALL` saved non-volatiles, FS, the exception-list head,
    and the caller's `PreviousMode`; replaced FS with `KGDT_R0_PCR`;
    re-set `Thread.PreviousMode` from the saved CS.
4.  `KiSystemServiceCopyArguments` copied `KiArgumentTable[svc]` bytes
    from the caller's stack onto the kernel stack (the canonical "rep
    movsd" loop).
5.  `call [KiServiceTable + svc*4]` ran the service.
6.  `KiServiceExit` checked for pending APCs, dispatched them if needed,
    then ran `EXIT_ALL` which restored everything via `IRETD`.

This is correct and uniform but each transition pays for the full IDT
gate — typically 200–300 cycles on modern silicon — even when the caller
was already in ring 0.

## 2. User-mode path: SYSENTER / SYSEXIT

`SYSENTER` (Pentium II, 1997) skips the IDT entirely and is the only
sensible choice for the user→kernel transition on any CPU MicroNT
realistically runs on (qemu-tcg, OVMF, post-2000 hardware). MicroNT
requires SEP and BugChecks if absent.

### 2.1 Per-CPU MSR programming

Three MSRs are programmed once per CPU by `KiSetupSysenter`
(`KE/I386/KERNLINI.C`), called near the end of `KiInitializeKernel`:

| MSR | Value |
|---|---|
| `IA32_SYSENTER_CS` (0x174) | `KGDT_R0_CODE` (0x08) |
| `IA32_SYSENTER_EIP` (0x176) | `&KiFastSystemCall` |
| `IA32_SYSENTER_ESP` (0x175) | initial idle thread `TSS.Esp0` |

`SYSENTER` hardcodes `SS = CS+8`, and `SYSEXIT` hardcodes the user
selectors at `CS+16` / `CS+24`. NT 3.5's GDT layout
(`R0_CODE=8, R0_DATA=16, R3_CODE=24, R3_DATA=32`) satisfies all four
with a single `IA32_SYSENTER_CS` write — no GDT reshuffling.

### 2.2 Per-thread MSR_SYSENTER_ESP reload

`SYSENTER` reads ESP directly from `MSR_SYSENTER_ESP`, *not* from the
TSS. The MSR therefore needs to track the current thread's kernel-stack
top across context switches.

Two sites update it:

-   `SwapContext` (`KE/I386/CTXSWAP.ASM` line 753 area) — the canonical
    context-switch path, alongside the existing `TSS.Esp0` write.
-   `Ki386AdjustEsp0` — invoked after V86/exception manipulation to
    realign `TSS.Esp0`; mirrors the same delta into the MSR.

Both sites use raw `0F 30` opcode bytes for `WRMSR` (ML 6.11d predates
the mnemonic).

`SwapContext`'s ordering matters: the MSR write must follow the TSS
write so that, in case an interrupt fires mid-switch, both stack
pointers point at the same destination thread's stack.

### 2.3 KiFastSystemCall entry

`KiFastSystemCall` (`KE/I386/TRAP.ASM`) runs at CPL0 with:

```asm
EAX = service number
EDX = pointer to caller's args            (lea edx, [esp+4] in the stub)
ECX = user ESP at sysenter                (mov ecx, esp in the stub;
                                           [ecx] = ret-to-caller-of-Nt*)
CS  = KGDT_R0_CODE                        (set by SYSENTER)
SS  = KGDT_R0_DATA                        (set by SYSENTER)
ESP = MSR_SYSENTER_ESP                    (top of current thread's K-stack)
EIP = MSR_SYSENTER_EIP                    (= &KiFastSystemCall)
EFLAGS = unchanged                        (SYSENTER does NOT clear IF)
```

Because SYSENTER does not push anything, the handler synthesises an
INT-2E-shaped trap frame so the existing `ENTER_SYSCALL` macro and the
shared dispatch body run unchanged:

```asm
cli                                 ; SYSENTER doesn't mask IF — close window
push KGDT_R3_DATA OR RPL_MASK       ; TsHardwareSegSs
push ecx                            ; TsHardwareEsp (raw user ESP — bumped below)
push EFLAGS_INTERRUPT_MASK          ; TsEflags = synthetic IF=1
push KGDT_R3_CODE OR RPL_MASK       ; TsSegCs (signals UserMode to ENTER_SYSCALL)
push dword ptr [ecx]                ; TsEip = [user_ESP] = ret-to-caller-of-Nt*
```

The deref of `[ecx]` reads from user memory at CPL0. If the user passed
a junk ESP this faults; we accept the fault as a deliberate user-error
signal (same fate as a junk `EDX` in the legacy INT 2E arg copy).

After ENTER_SYSCALL builds the rest of the frame, the body bumps
`TsHardwareEsp` by `4 + ArgSize` — the kernel emulates the user stub's
`ret 4*NumArgs` so the caller-of-`Nt<Func>` resumes with a balanced
stack and never re-enters the per-stub `ret`.

### 2.4 KiServiceFastExit (SYSEXIT)

Eligibility for the SYSEXIT fast return is checked at the top of
`KiServiceExit`. All of:

-   `TsSegCs` is canonical user (`KGDT_R3_CODE | RPL=3`).
-   `TsHardwareSegSs` is canonical user (`KGDT_R3_DATA | RPL=3`).
-   `TsEflags & (TF | VM) == 0` — no single-step, no V86 return.
-   `Thread.DebugActive == 0` — DR0–DR7 unused; skip the `EXIT_ALL`
    `Dr_ExitHelp` step.

If any fails, the legacy `EXIT_ALL`/`IRETD` path runs. Otherwise:

```asm
mov edx, [esp]+TsExceptionList         ; restore PCR.ExceptionList
mov fs:[PcExceptionList], edx
mov ecx, [esp]+TsPreviousPreviousMode
mov [thread]+ThPreviousMode, cl
mov edx, [ebp]+TsEip                   ; sysexit EIP
mov ecx, [ebp]+TsHardwareEsp           ; sysexit ESP (bumped past args by entry)
lea esp, [ebp]+TsSegFs                 ; restore FS
pop fs
pop edi / esi / ebx / ebp
sti                                    ; user expects IF=1
db 0Fh, 35h                            ; SYSEXIT
```

`SYSEXIT` does not restore EFLAGS; NT's calling convention treats EFLAGS
as caller-save across function calls (DF=0 on entry, arithmetic flags
scratch), so the only user-visible loss is `TF` — gated by the
eligibility check above.

### 2.5 Round-trip cost

`NtQuerySystemTime` round-trip under qemu-tcg measured at ~0.5 µs/call
post-cutover (vs. ~3-5 µs/call on the INT 2Eh baseline). Most of the
saving comes from skipping the IDT-gate trap-frame ceremony; the
SYSEXIT fast-return contributes the remaining ~½ of the win.

## 3. Kernel-mode path: `_KiKernelDispatch`

Kernel-mode `Zw*` callers — drivers calling `ZwOpenKey` for registry,
`ZwReadFile` for file I/O, the HAL's `halps2.c` / `ixpcibrd.c` /
`ixpcibus.c` registry probes — never see the trap mechanism. The
`SYSSTUBS_ENTRY1` macro (`KE/I386/sysstubs.asm`) emits, for each `Zw*`:

```asm
mov eax, ServiceNumber
jmp _KiKernelDispatch
```

`_KiKernelDispatch` (defined just above the SYSSTUBS_ENTRY invocations
in the same file) builds a per-call frame, copies the args once,
dispatches, restores `PreviousMode`, and returns directly to the caller
of `Zw<Func>`.

### 3.1 Why a small arg copy

A truly zero-copy version was considered: stash the caller's
`PreviousMode` in a per-thread scratch slot (KTHREAD has slack between
`ThreadObjectLength = 0x1B0` and `ExtendedThreadObjectLength = 0x240`)
and chain a trampoline that the service's `ret 4*N` lands on.

This breaks under recursion. Driver code inside one `Zw*` regularly
issues another `Zw*`; the inner dispatch's scratch-slot write stomps the
outer call's saved state. The legacy INT-2Eh path supports arbitrary
recursion implicitly because every trap allocates a fresh kernel-stack
frame.

The simplest direct-dispatch design that preserves that property is to
do the same: small per-call kernel-stack frame, copy the args once,
restore on return. The arg copy is `ArgSize / 4` `movsd` cycles — single
digits for almost every service — and is dwarfed by the saving of *not*
doing IDT gate + ENTER_SYSCALL + EXIT_ALL.

### 3.2 Helper structure

```asm
_KiKernelDispatch:
    cmp eax, _KiServiceLimit ; defensive — gensrv guarantees in-range
    ja  kkd_bad_service       ; int 3

    push ebp
    mov  ebp, esp             ; [ebp+0]=saved_ebp, [ebp+4]=ret-to-caller,
                              ; [ebp+8..]=caller's args
    push ebx
    push esi
    push edi
    sub  esp, 4               ; [ebp-16] = ArgSize local (survives the call)

    movzx ecx, byte ptr _KiArgumentTable[eax]
    mov   [ebp-16], ecx
    mov   edx, _KiServiceTable[eax*4]                     ; service routine

    mov   edi, fs:[PcPrcbData+PbCurrentThread]            ; thread (preserved)
    movzx ebx, byte ptr [edi+ThPreviousMode]              ; old prev-mode (preserved)
    mov   byte ptr [edi+ThPreviousMode], 0                ; KernelMode

    sub   esp, ecx                                        ; allocate arg copy
    lea   esi, [ebp+8]                                    ; src = caller's args
    mov   edi, esp                                        ; dst = our allocation
    shr   ecx, 2
    rep   movsd

    mov   edi, fs:[PcPrcbData+PbCurrentThread]            ; movsd trashed edi
    call  edx
    ; Service ret 4*N popped its arg copy + ret-addr.
    ; ESP is back to ebp-16 (local). EAX = NTSTATUS.
    ; EDI/EBX preserved across the call.

    mov   byte ptr [edi+ThPreviousMode], bl               ; restore prev-mode
    mov   ecx, [ebp-16]                                   ; ArgSize for callee-pop
    add   esp, 4
    pop   edi / esi / ebx / ebp

    pop   edx                                             ; ret-to-caller
    add   esp, ecx                                        ; pop original args
    jmp   edx
```

Register protocol relies on the stdcall ABI: service routines preserve
`EBX/ESI/EDI/EBP`, so we use `EDI` to carry the thread pointer and `EBX`
to carry the saved `PreviousMode` across the call without saving them
ourselves.

### 3.3 What we don't do

-   **No trap frame.** No FS reload to PCR (CPL0 caller already has
    PCR-FS), no IRQL-elevated debug check.
-   **No `ENTER_SYSCALL` / `EXIT_ALL`.** No DR-assist branch, no V86
    branch, no abios branch — none of these are reachable from a
    kernel-mode caller.
-   **No IDT vector for INT 2Eh.** Vector 0x2E is invalidated; any stray
    `INT 2Eh` raises `#GP` and is loud about it.

### 3.4 SEH chain handling on the dispatch boundary

`_KiKernelDispatch` saves `PCR.ExceptionList` and writes
`EXCEPTION_CHAIN_END` over it before calling the service, then restores
the saved value on the normal-return path.  This mirrors what
`ENTER_SYSCALL` did inside the legacy INT-2Eh trap frame.

The contract this enforces — by NT 3.5 design and by inheritance here:

> Kernel-mode `Zw*` is a return-NTSTATUS interface.  A service routine
> that raises an exception out through a kernel-mode `Zw*` call is a
> contract violation.  When it happens the kernel bug-checks with
> `KMODE_EXCEPTION_NOT_HANDLED`.

The mechanism: with `PCR.ExceptionList = EXCEPTION_CHAIN_END`,
`RtlDispatchException` finds no handler when the service raises and
delivers an unhandled-exception bug-check.  Bug-checking is what we
want — the alternative ("let the exception unwind into the caller's
chain") would silently leave `Thread.PreviousMode` stuck at
`KernelMode`, since `_KiKernelDispatch`'s prev-mode-restore epilogue is
unreachable on the unwind path.  Subsequent probes anywhere in the
unwound caller would over-trust user pointers.

A future revision could install a proper `_SEH_prolog`-style
registration whose handler restores `PreviousMode` (and the rest of
the WRK-era `KiSystemServiceHandler` undo set) and continues the
search — that would let drivers wrap `Zw*` in `__try` without a bug
check.  Today no driver in the tree does so, and the bug-check
contract is the simplest, loudest, and most forward-debuggable
behaviour.

The audit that justifies this:

-   C-side kernel `Zw*` callers: 3 files in HAL
    (`halps2.c`, `ixpcibrd.c`, `ixpcibus.c`), all registry calls, all
    NTSTATUS-checked, **zero `_try` blocks**.
-   Asm-side kernel callers: `EX/I386/RAISE.ASM` uses `_ZwContinue` and
    `_ZwRaiseException` as control-flow primitives that don't raise
    out of themselves (the service patches a trap frame and returns
    success).
-   The legacy `Kss_ExceptionHandler` in `TRAP.ASM` was likewise
    designed to bug-check on kernel-mode service raise — currently
    `if 0`'d, but the underlying `EXCEPTION_CHAIN_END` mechanism in
    `ENTER_SYSCALL` produced the same bug-check via the unhandled
    chain walk.

## 4. Compatibility surface

Driver source compatibility — unchanged. `Zw*` exports keep their
stdcall ABI bit-for-bit; `KTHREAD` is opaque in `ntddk.h`; no documented
field offsets shift; no struct sizes change.

User-mode binary compatibility — unchanged. `Nt*` exports in ntdll keep
their stdcall ABI; the macro body changes from `INT 2Eh` to
`mov ecx, esp; sysenter` but the function signature, name, and
calling convention stay identical. User-mode debuggers that walk the
stack via frame pointers see the same frames as before.

The one observable difference: a single-stepping user-mode debugger
loses TF across a syscall (SYSEXIT discards `EFLAGS`). MicroNT relies
on qemu+gdb for kernel-mode debugging, so this is an accepted trade.
